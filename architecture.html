Stack tecnico

Cada componente elegido por rendimiento, simplicidad y escalabilidad

🌐

Frontend — Vanilla JS PWA

HTML unico + JS puro. Sin frameworks, sin build tools. IndexedDB + localStorage para offline. Service Worker con cache versionado (v11). Chart.js, Leaflet, MQTT.

⚡

Backend — FastAPI + SQLAlchemy

Python 3.12, FastAPI 0.115 async, SQLAlchemy 2.0 con asyncpg. Pool de 20+30 conexiones. JSON structured logging. Sentry para error tracking.

🗄️

Base de datos — PostgreSQL 16

26 tablas multi-tenant con UUIDs. Alembic para migraciones (11 versiones). Indices en organization_id, timestamps y FKs criticas. Backups automaticos en Docker.

🔴

Cache — Redis 7

Rate limiting por IP (120 req/min). Cache de sesiones. Sliding window con INCR + EXPIRE. Fail-open: si Redis cae, las requests pasan sin bloqueo.

💳

Pagos — Stripe

Checkout sessions, customer portal, webhooks para lifecycle. Descuentos por fase: trial → 40% → 25% → 15% → precio completo. Soft landing para conversion.

📧

Email — Resend API

Verificacion, reset de password, invitaciones de equipo, bienvenida. HTML templated. Archivo local de cada email enviado en PostgreSQL.

🐳

Infra — Docker Compose

4 servicios: app (FastAPI + Uvicorn), PostgreSQL 16, Redis 7, backup automatico. VPS unico en api.egglogu.com. CDN Cloudflare Pages para frontend.

🔒

Auth — JWT + OAuth multi

JWT con refresh tokens. Google OAuth, Apple Sign-In, Microsoft Identity. PIN offline para acceso sin internet. Roles: superadmin, owner, manager, vet, viewer.

🌍

i18n — 8 idiomas

ES, EN, PT, FR, DE, IT, JA, ZH. Sistema de traducciones con 200+ keys. Deteccion automatica del navegador. Cambio en tiempo real sin reload.

Seguridad en cada capa

Desde el navegador hasta la base de datos, cada punto esta protegido

🔒

JWT + Refresh Tokens

Access token de corta vida + refresh token rotativo. Bcrypt para hashing. Blacklist en Redis para logout instantaneo.

🛡️

Content Security Policy

CSP estricto: solo CDNs aprobados para scripts. SHA-256 hashes para estilos inline. Bloquea XSS por inyeccion de scripts.

⏱️

Rate Limiting multi-capa

120 req/min global por IP. Limites especificos por endpoint de auth. Rate limit por usuario y organizacion. Redis-backed con fail-open.

🏠

Multi-tenancy estricto

Cada query filtra por organization_id. Imposible acceder a datos de otro tenant. FK con CASCADE para limpieza automatica.

📜

Security Headers

HSTS (1 ano), X-Frame-Options: DENY, X-Content-Type: nosniff, Referrer-Policy: strict-origin. Previene clickjacking y MIME sniffing.

🔍

Audit Log

Registro de acciones criticas: CREATE, UPDATE, DELETE. IP, User-Agent, diff de cambios en JSON. Para compliance y forense.

📴

Offline PIN seguro

PIN para acceso sin internet, hasheado localmente. No se transmite al servidor. Expiracion automatica.

💾

Datos locales cifrados

Datos sensibles en localStorage protegidos con Web Crypto API. Clave derivada del password del usuario. AES-GCM 256-bit.

Sincronizacion delta

Offline-first con resolucion de conflictos inteligente

1

Captura offline

Datos se escriben en IndexedDB + localStorage. Marca timestamp de cada cambio.

2

Deteccion de red

El Service Worker detecta reconexion. Acumula cola de cambios pendientes.

3

Push + Pull

POST /sync envia cambios locales + last_synced_at. Server responde con cambios remotos.

4

Merge inteligente

Last-write-wins por campo. Conflictos notificados al usuario. Cursor para paginacion.
